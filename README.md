# RoboCon Timer 2026

ロボット競技（ロボコン）向けの**完全遅延ゼロ・リアルタイムタイマー＆スコアボードシステム**です。

表示にブラウザの重いレンダリングを使わず、**タイマーの心臓部（ステートマシン）を Python サーバー側**に置くことで、操作画面・表示画面間のネットワーク遅延やブラウザの処理落ちによるカクツキ・ズレを完全に排除しました。

---

## 📁 ディレクトリ構造

ファイル類は整理され、静的アセットはすべて `public/` フォルダ内に格納されています。

```
/robocontimer2026
├── README.md               # このファイル
├── requirements.txt        # インストール必要なパッケージリスト
├── server.py               # サーバー＆タイマー心臓部（ステートマシン）
├── display.py              # 低遅延・Pythonネイティブのタイマー表示画面
│
├── public/                 # 静的ファイル群（ブラウザ画面用）
│   ├── index.html          # 大会運営用コントロール画面
│   ├── screen.html         # プロジェクター用 フル表示画面 (ブラウザ版)
│   ├── overlay.html        # OBS配信合成用 クロマキー画面 (ブラウザ版)
│   ├── css/                # スタイルシート
│   ├── js/                 # 操作パネル用 (コントロール・音声等)
│   └── data/               # ★ チーム名・スコアルール設定ファイル ★
│
└── webrtc_poc/             # 【実験機能】スマホカメラ配信用 PoC
```

## 🌟 主な機能

| 機能 | 説明 |
|---|---|
| **遅延ゼロ同期 (新機能)** | タイマーの「1秒ごとの計算」をサーバー側(Python)で行い配信します。コントロール側が処理落ちしてもタイマーは正確に進み続けます |
| タイマー | セッティングタイム（カウントダウン）/ マッチタイム（カウントアップ） |
| カウントダウン演出 | スタート前の 5→4→3→2→1→START シーケンス |
| スコアボード | 赤・青チームの得点を JSON 設定でカスタマイズ |
| 自動機/手動機モード | 同一アクションでも機種別に得点配分を変更可 |
| V-GOAL 演出 | 黄色フラッシュ + タイマー自動停止 |
| 音響効果 | カウントダウンピッ音 / スタート＆終了ブザー音 |
| キーボードショートカット | キー 1 回で得点加算、Shift + キーで減算 |
| スマホ対応 | コントロール画面はレスポンシブデザイン |

## 🚀 起動方法

### 1. サーバー（タイマー心臓部）を起動する

```bash
python server.py
```

`http://localhost:8080/` でサーバーが起動し、タイマーエンジンが活動を開始します。

### 2. コントロール画面を開く（ブラウザ必須）

```
http://localhost:8080/
```

タイマー操作・得点入力はここから行います。<br>スマホからも操作できます（同一 Wi-Fi 内で `http://[サーバーのIP]:8080/` へアクセス）。
※ 操作パネルは計算を行わず、サーバーへ命令を送るだけのリモコンとして動作します。

## 📺 表示画面の選択

### 方法A：Python ネイティブウィンドウ（推奨・超低遅延）

ブラウザを使わないため、OBS のウィンキャプチャ等で取り込むと**ブラウザソースよりはるかに低遅延・低負荷**になります。  
また、Web Audio API の制限を受けないため、初回のクリックも不要で音声が確実に鳴ります。

#### フル表示（プロジェクター用）

```bash
python display.py
```

| 操作 | キー |
|---|---|
| フルスクリーン切替 | `F` |
| フルスクリーン解除 | `Escape` |
| 終了 | `Ctrl + Q` |

**OBS 取り込み方法**:  
ソース追加 → 「ウィンドウキャプチャ」→ `ROBOCON TIMER || Full Display` を選択

#### OBS オーバーレイ用（クロマキー合成用）

```bash
python display.py --overlay
```

「チーム名＋スコア」を左右に、「タイマー」を中央に配置する横長バー型レイアウトです。背景色が `#00ff00`（極緑）になり、クロマキーで切り抜くことができます。

**OBS 取り込み方法**:
1. ソース追加 → 「ウィンドウキャプチャ」→ `ROBOCON TIMER || Overlay`
2. そのソースを右クリック → **フィルタ** → `+` → **「クロマキー」** を追加
3. キーカラータイプ: `緑` / 類似度: `1〜5` に設定

#### 別の PC で動いているサーバーを指定する

```bash
python display.py --server http://192.168.1.100:8080
```

---

### 方法B：ブラウザ版（フォールバック用）

Pythonがインストールされていない別PCでの表示や、旧環境向けです。
サーバー起動後、ブラウザで以下の URL を開きます。  
※ 初回のみ、音声を有効にするため画面のどこかを1度クリックしてください。

| 画面 | URL |
|---|---|
| プロジェクター用 | `http://localhost:8080/screen.html` |
| OBS オーバーレイ用 | `http://localhost:8080/overlay.html` |

## ⚙️ 大会ルール / 設定ファイル

`public/data/` 内部の JSON ファイルを編集することで、リアルタイムにシステムルールの変更ができます。

### 1. チーム名リスト (`public/data/teams.json`)

```json
["TEAM ALFA", "TEAM BRAVO", "TEAM CHARLIE"]
```

コントロール画面のドロップダウンに反映されます。

### 2. 得点ルール (`public/data/scores.json`)

```json
[
    {
        "id": "ring",
        "label": "リング",
        "points": { "manual": 10, "auto": 20 },
        "type": "number",
        "keyRed": "a",
        "keyBlue": "l"
    },
    {
        "id": "vgoal",
        "label": "Vゴール",
        "points": 0,
        "type": "toggle",
        "keyRed": "e",
        "keyBlue": "o"
    }
]
```

- **`points`**: 手動/自動で異なる場合は `{ "manual": 10, "auto": 20 }`。同じなら単に `10` と書く。
- **`type`**: `number`（加算方式カウント） か `toggle`（ON/OFF 切替式）。
- **`keyRed` / `keyBlue`**: 割り当てるキーボードショートカット。**Shift** と同時押しで減算が行えます。

## 🔊 音声について

| タイミング | 波形仕様（ブラウザ・Pythonネイティブ完全一致） |
|---|---|
| カウントダウン 3・2・1 | 500 Hz 三角波 単一オシレーター（0.25 秒） |
| START / 試合終了時 | 1000 Hz の Sawtooth(×2) + Square 混合ブザー（0.8 秒） |

画面が複数開いている場合も、同じタイミングでの重複再生(2重鳴り)は防止されます。

## 📡 アーキテクチャの概要

旧バージョンからの変更点：  
以前は `control.js` が時間を刻み巨大なステートを更新していましたが、「タイマーの処理落ちや通信ごとの細かな遅延」を排すため、役割を整理しました。

```
[操作用 index.html]  ── POST /api/command ──> [ Python server.py ] (タイマー親玉)
  (↑ 操作はただの         {"cmd": "start"}             │
   リモコンになった )                                   │ "1秒ごとに正確にTick"
                                                       │
                                                       └─> SSE ブロードキャスト
                                                              ├─> [プロジェクター display.py]
                                                              ├─> [OBS overlay display.py]
                                                              └─> [操作用 index.html にもフィードバック]
```

この構成により、Wi-Fi環境等で操作PCの通信が一瞬詰まってもタイマー表示はストップしません。

## 📷 【実験機能】スマホカメラ WebRTC 配信 (`webrtc_poc/`)

スマホのカメラ映像を低遅延の WebRTC 経由で取得し、OBS の「メディアソース」用に MJPEG 変換して配信します。

```bash
cd webrtc_poc
python generate_cert.py    # 初回のみ（HTTPS 証明書生成）
python server.py
```
表示されたURL（例: `https://192.168.x.x:8443/`）にスマホのブラウザからアクセスして映像を送信。OBS 側ではメディアソース（ローカルファイルのチェック外す）に `http://[サーバIP]:9001/stream` を指定（ネットワークキャッシュ: 0）。
